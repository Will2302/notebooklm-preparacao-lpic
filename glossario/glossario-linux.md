# Glossário de Conceitos Linux (LPIC‑1 / LPIC‑2)

- **Shell**: interpretador de comandos que permite interagir com o sistema Linux via terminal.
- **Path (caminho)**: localização de um arquivo ou diretório no sistema de arquivos (ex.: `/home/usuario/arquivo.txt`).
- **Permissão de arquivo**: controle de quem pode ler, escrever ou executar um arquivo, usando `chmod`, `chown`, etc.
- **Partição MBR**: esquema de particionamento antigo, limitado a 4 partições primárias e 2 TB por disco.
- **GPT**: esquema de particionamento moderno, suporta mais partições e discos maiores.
- **Sistema de arquivos**: método de organização de dados em disco (ex.: ext4, XFS).
- **Swap**: espaço em disco usado como memória virtual para o sistema.
- **Runlevel**: sistema de inicialização legado (SysVinit) usado em alguns temas de LPIC.
- **Systemd**: sistema moderno de inicialização e gerenciamento de serviços.
- **Serviço (service)**: processo em background que executa tarefas contínuas (ex.: SSH, cron, Apache).
